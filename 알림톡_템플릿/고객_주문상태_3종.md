# 알림톡 템플릿 — 손님 주문상태 3종 (예약 모델)

> 2026-06-28 예약 모델 전환에 따른 손님 알림톡 재정리.
> 손님이 결제화면에서 **픽업 시간을 직접 선택**(예약)하고, 점주가 **확정 / 시간변경 / 거절** 한다.
> 발송 트리거: `owner.orders.tsx` updateStatus 액션 → `N8N_WEBHOOK_URL`(고객 워크플로 `kakao_order_customer_sms` / n8n `3OdyLnA9plF1gBRv`, path `/webhook/kakao_customer`).
> **n8n은 payload `notificationType` 으로 아래 3개 템플릿을 분기**한다.

## 웹훅 payload (점주 액션 시) — 분기 키 + 변수 소스
```jsonc
{
  "notificationType": "confirmed" | "changed" | "rejected",  // ← n8n 분기 키
  "cancelReason": "재료 소진으로 주문이 어렵습니다" | null,     // rejected 일 때만
  "order": {
    "phoneNumber": "01012345678",          // 손님 수신번호
    "totalAmount": 21000,
    "requestedPickupTime": "2026-06-28T09:00:00.000Z",  // 손님이 요청했던 시각(ISO)
    "estimatedPickupTime": "2026-06-28T09:30:00.000Z"   // 점주가 확정한 시각(ISO)
  },
  "items": [{ "menuName": "짬뽕", "options": ["곱빼기"], "quantity": 2, ... }],
  "store": { "storename": "굿모닝차이나", ... }
}
```
> 시각은 ISO(UTC) → n8n Build 노드에서 **KST "오후 N시 M분"** 으로 변환(기존 손님 워크플로 변환 로직 재사용).

---

## ① 픽업 확정 — `notificationType: "confirmed"`
점주가 손님 요청시간 **그대로 수락**. (기존 `UI_8730` "주문접수_고객" 거의 그대로 재사용 가능)

```
[#{가게명}] 주문이 확정되었습니다 ✅

요청하신 시간에 픽업 준비를 시작합니다.

▶ 픽업 시간: #{픽업시간}
▶ 주문내역: #{주문내역}
▶ 결제예정금액(현장): #{결제금액}원

시간에 맞춰 방문해 주세요. 결제는 현장에서 진행됩니다.
```
| 변수 | 소스 |
|---|---|
| 가게명 | `store.storename` |
| 픽업시간 | `order.estimatedPickupTime`(=요청시간) → KST |
| 주문내역 | `items[].menuName × quantity` (+옵션) |
| 결제금액 | `order.totalAmount` 콤마 |

---

## ② 픽업 시간 변경 — `notificationType: "changed"`
점주가 요청시간을 **다른 시간으로 바꿔서** 수락. (신규 템플릿)

```
[#{가게명}] 픽업 시간이 변경되었습니다 🕒

사장님 사정으로 픽업 시간이 아래와 같이 조정되었습니다.

▶ 변경된 픽업 시간: #{변경시간}
▶ 주문내역: #{주문내역}
▶ 결제예정금액(현장): #{결제금액}원

변경된 시간에 맞춰 방문해 주세요. 결제는 현장에서 진행됩니다.
```
| 변수 | 소스 |
|---|---|
| 가게명 | `store.storename` |
| 변경시간 | `order.estimatedPickupTime`(점주 확정) → KST |
| 주문내역 | `items[]` |
| 결제금액 | `order.totalAmount` |
> (선택) 본문에 `요청 #{요청시간} → 변경 #{변경시간}` 비교를 넣고 싶으면 `order.requestedPickupTime`도 변수로 추가.

---

## ③ 주문 거절 — `notificationType: "rejected"`
점주가 주문을 거절(CANCEL). 사유는 점주가 드롭다운에서 선택. (신규 템플릿)

```
[#{가게명}] 주문을 받지 못했습니다 🙏

죄송합니다. 아래 사유로 이번 주문은 접수되지 못했습니다.

▶ 사유: #{거절사유}
▶ 주문내역: #{주문내역}

불편을 드려 죄송합니다. 다음에 다시 찾아주세요.
```
| 변수 | 소스 |
|---|---|
| 가게명 | `store.storename` |
| 거절사유 | `cancelReason` (점주 선택값, 4종 중 1) |
| 주문내역 | `items[]` |

**거절 사유 4종**(코드 `owner.orders.tsx` `CANCEL_REASONS`):
- 재료 소진으로 주문이 어렵습니다
- 주문이 많아 시간 내 준비가 어렵습니다
- 영업 종료(마감)로 주문이 어렵습니다
- 요청하신 픽업 시간에 맞추기 어렵습니다

---

## 반려 방지 체크 (3종 공통)
- **정보성**으로 분류(광고 단어 금지: 이벤트/할인/쿠폰 X).
- 고정 안내문 > 변수 비율 유지.
- 버튼 불필요(손님은 별도 화면 없음). 필요 시 "주문 내역 확인" 웹링크만.

## 카카오 승인 후 n8n 작업
1. 3개 템플릿 각각 승인 → tpl_code 3개 확보(예: `UI_xxxx`).
2. 손님 워크플로 `3OdyLnA9plF1gBRv` 에 **`notificationType` Switch** 추가 → 3분기.
3. 각 분기에서 해당 tpl_code + 변수 치환(기존 confirmed=UI_8730 변환로직 재사용, KST 변환 공통).
4. 발송 실패 시 SMS 폴백은 기존 패턴 유지.
