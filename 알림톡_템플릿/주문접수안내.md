# 알림톡 템플릿 — 주문접수안내 (1순위 · 무조건 필요)

> 카카오 비즈니스 채널 → 알림톡 → 템플릿 관리 → 새 템플릿 에 아래 내용 등록.
> 발송 시점: 사장님이 주문 ACCEPT 할 때 (코드: `app/routes/owner.orders.tsx:175` 웹훅).

## 등록 설정
- **템플릿명:** `주문접수안내`
- **메시지 분류:** 정보성 (주문/예약)
- **메시지 유형:** 기본형
- **버튼:** 없음 (나중에 추가 가능)

## 템플릿 내용 (실제 등록본 — 2026-06-22 테스트 등록)

```
[#{가게명}] 주문이 접수되었습니다

고객님, 주문해주셔서 감사합니다.
아래 내용으로 주문이 정상 접수되었습니다.

▶ 주문번호: #{주문번호}
▶ 주문내역: #{주문내역}
▶ 예상 결제금액(현장): #{결제금액}원
▶ 픽업 예정시간: #{픽업시간}

문의: #{가게전화}
```

> 변경: `#{고객명}` 변수 제거 → "고객님" 고정(닉네임 의존 없앰), 안내 2줄(준비완료/취소불가) 제거.

## 변수 (6개)
| 변수 | 미리보기값(심사용) | 코드 매핑 (owner.orders.tsx payload) |
|---|---|---|
| `가게명` | 굿모닝차이나 | `store.storename` |
| `주문번호` | A1024 | `orderId` |
| `주문내역` | 짬뽕 1, 짜장면 2 | `items[].menuName` + 옵션 조합 |
| `결제금액` | 21000 | `order.totalAmount` |
| `픽업시간` | 오후 6시 30분 | `order.estimated_pickup_time` |
| `가게전화` | 032-327-9696 | `store.storenumber` |

## 반려 방지 체크
- 정보성으로 분류 (광고 단어 금지: 이벤트/할인/쿠폰 X)
- 고정 안내문이 변수보다 많아야 통과 — 이 템플릿은 충족
- 변수 미리보기값은 위 표처럼 실제처럼 입력

## 더 단순하게 가려면 (선택)
`#{고객명}님` → `고객님` 으로 바꾸면 변수 6개로 등록 가능.

## 승인 후 할 일
승인되면 **템플릿코드**가 나옴 → 그 코드로 n8n HTTP 노드를 SMS → 알림톡 엔드포인트로 교체 (SMS는 폴백 유지).
