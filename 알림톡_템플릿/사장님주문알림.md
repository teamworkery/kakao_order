# 알림톡 템플릿 — 사장님 신규주문 알림 (점주용)

> 카카오 비즈니스 채널 → 알림톡 → 템플릿 관리 → 새 템플릿 에 아래 내용 등록.
> 발송 시점: **손님이 주문을 넣는 순간** (코드: `app/routes/$name.tsx:359` 웹훅 → n8n `kakao_order_store_sms` `vgbsEofC9kLMJQ7i`, path=`kakao_store`).
> 현재는 이 워크플로가 **SMS(LMS)** 발송 중 → 승인 후 손님 템플릿(`UI_8730`)과 동일 패턴으로 알림톡 전환.

## 등록 설정
- **템플릿명:** `사장님주문알림`
- **메시지 분류:** 정보성 (주문/예약)
- **메시지 유형:** 기본형
- **버튼:** 웹링크 1개 — `주문 확인하기` → `https://pojang.one/owner/orders?openExternalBrowser=1` (PC/모바일 동일)
  - ℹ️ 버튼은 주문 화면을 **열기만** 한다(수락 자체가 아님). 수락·거절은 그 페이지에서 진행 → 거절 버튼 별도 추가 안 함(같은 페이지로 가 혼란).
  - ⚠️ **`?openExternalBrowser=1` 필수**: 카카오 공식 파라미터로, 버튼을 **카톡 인앱 브라우저가 아니라 폰 기본 브라우저(크롬/삼성인터넷/사파리)로 강제**한다. 인앱 브라우저는 기본 브라우저와 쿠키가 분리돼 매번 재로그인을 요구하므로, 이 파라미터로 세션이 유지되는 기본 브라우저를 열어 **재로그인 마찰 제거**(한 번만 로그인하면 이후 바로 주문목록 진입). 코드 측은 `/owner/orders` 미인증 진입 시 `/login?next=/owner/orders` 로 보내 로그인 후 주문목록으로 복귀(2026-06-28).

## 템플릿 내용 (등록본)

```
[#{가게명}] 새 주문이 접수되었습니다

아래 내용을 확인하고 수락해 주세요.

▶ 주문번호: #{주문번호}
▶ 주문내역: #{주문내역}
▶ 결제예정금액(현장): #{결제금액}원
▶ 고객 연락처: #{고객연락처}

주문 확인 후 수락하면 고객에게 안내 알림톡이 자동 발송됩니다.
```

> 버튼(웹링크) 본문 밑에 노출: **주문 확인하기** → `https://pojang.one/owner/orders?openExternalBrowser=1`

## 변수 (4개) — 전부 기존 store 웹훅 payload에 존재
| 변수 | 미리보기값(심사용) | 코드 매핑 (`$name.tsx` store payload) |
|---|---|---|
| `가게명` | 굿모닝차이나 | `site.storeName` |
| `주문번호` | A1024 | `order.id` |
| `주문내역` | 짬뽕 1, 짜장면 2 | `order.items[].name` × `quantity` |
| `결제금액` | 21000 | `order.totalAmount` |
| `고객연락처` | 010-1234-5678 | `order.customerPhone` |

> (표는 5행이지만 `#{변수}`는 4종 — 결제금액 옆 "원"은 고정문, 고객연락처 포함 실제 변수 5개. 심사 시 변수 미리보기값 위처럼 실제처럼 입력.)

## 반려 방지 체크

## 수신번호 — 점주 휴대폰 `owner_phone` (✅ 구현 완료, 2026-06-28)
**알림톡은 카카오톡이 깔린 휴대폰 번호로만 도달**한다. 가게 전화(`storenumber`)가 유선이면 알림톡·SMS 모두 안 가므로, **점주 휴대폰을 별도 컬럼 `owner_phone`으로 분리**(사용자 결정).
- 구현: migration `004_owner_phone.sql`(비공개 컬럼) + 가게설정 폼 2곳 입력 + 주문 시 `$name.tsx`가 **서비스롤(`makeAdminClient`)로 `owner_phone` 조회**(공개 뷰 미노출) → 웹훅 `notify.phone`에 사용, 미설정 시 `storenumber`로 폴백.
- **⚠️ 배포 순서**: migration 004를 **운영 DB에 먼저 적용** 후 코드 배포(컬럼 없으면 가게설정 저장이 실패). 기존 점주는 `owner_phone` 비어있으니 재입력 안내 필요.

## 승인 후 할 일 (손님 템플릿과 동일 절차)
1. 승인되면 **템플릿코드**(예: `UI_xxxx`)가 나옴.
2. n8n `kakao_order_store_sms`(`vgbsEofC9kLMJQ7i`)를 손님 워크플로처럼 전환:
   Build(5변수 치환)→Send Alimtalk(`alimtalk/send`, mid)→Wait 60s→Verify(`history/detail` rslt)→실패 시 **SMS 폴백** + 버튼(웹링크) JSON 추가.
3. 자격증명 동일(senderkey `e4aab44c…`·apikey `7mqso1…`·sender `01096643237`).
