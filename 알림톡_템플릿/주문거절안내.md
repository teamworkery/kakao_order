# 알림톡 템플릿 — 주문 거절 안내 (고객용 · 신규 등록)

> 2026-06-29 확정. 시간변경은 보류, **거절은 필수 기능**으로 등록한다.
> 발송 시점: 점주가 주문을 **거절(CANCEL)** 할 때. 코드: `app/routes/owner.orders.tsx` updateStatus → `N8N_WEBHOOK_URL`(고객 워크플로 `kakao_order_customer_sms` / n8n `3OdyLnA9plF1gBRv`).
> 거절 알림은 n8n `notificationType:"rejected"` 분기 + Vercel env `ENABLE_CUSTOMER_REJECT_NOTIFY=true` 가 켜져야 실발송된다(기본 OFF=오발송 방지).

## 등록 설정
- **템플릿명:** `주문거절안내`
- **메시지 분류:** 정보성 (주문/예약)
- **메시지 유형:** 기본형
- **버튼:** 없음

## 템플릿 내용 (등록본)

```
[#{가게명}] 주문을 받지 못했습니다

죄송합니다. 아래 사유로 이번 주문은 접수되지 못했습니다.

▶ 사유: #{거절사유}
▶ 주문내역: #{주문내역}

불편을 드려 죄송합니다. 다음에 다시 찾아주세요.

문의: #{가게전화}

```

## 변수 (4개) — 심사용 미리보기값
| 변수 | 미리보기값(심사용) | 코드 매핑 |
|---|---|---|
| `가게명` | 굿모닝차이나 | `store.storename` |
| `거절사유` | 재료 소진으로 주문이 어렵습니다 | `cancelReason` (점주 선택값, 4종 중 1) |
| `주문내역` | 짬뽕(곱빼기) 2*8,000원, 짜장면 1*6,000원 | n8n Build (수량*단가원) |
| `가게전화` | 032-327-9696 | `store.storenumber` |

**거절 사유 4종**(코드 `owner.orders.tsx` `CANCEL_REASONS`):
- 재료 소진으로 주문이 어렵습니다
- 주문이 많아 시간 내 준비가 어렵습니다
- 영업 종료(마감)로 주문이 어렵습니다
- 요청하신 픽업 시간에 맞추기 어렵습니다

## 반려 방지 체크
- 정보성 분류 (광고 단어 금지: 이벤트/할인/쿠폰 X)
- 고정 안내문 > 변수 비율 — 충족
- 이모지 없이 등록(심사 안전). 필요 시 첫 줄 끝에 추가 가능.

## 승인 후 할 일
1. 승인 → tpl_code(예: `UI_xxxx`) 확보.
2. 고객 워크플로 `3OdyLnA9plF1gBRv` 에 `notificationType` Switch 추가 → confirmed(`UI_8730`) / rejected(이 코드) 분기.
3. Vercel env `ENABLE_CUSTOMER_REJECT_NOTIFY=true` 로 거절 통지 ON.
