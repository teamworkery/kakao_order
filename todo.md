# 개선사항 TODO 리스트

이 문서는 필수 기능만 포함하는 단순한 포장 주문 앱을 위한 개선사항을 기록합니다.

**중요 정책**:

- 이 앱은 **포장 주문 전용**이며, 결제 기능은 포함되지 않습니다
- 앱은 **단순성을 유지**하며 필수 기능만 포함합니다
- 주문 요청 후 **취소는 불가능**합니다
- 포장이므로 **픽업 가능 시간**을 주문 시점부터 명확히 표시해야 합니다

## 데이터베이스 스키마 개선

### Enum 확장

- [ ] `kakao_order` enum에 상태 추가:
  - [ ] `PREPARING` (조리 중)
  - [ ] `READY` (준비 완료)
  - [ ] `COMPLETED` (완료)
  - [ ] `CANCEL` 상태 제거 (취소 기능 없음)

### orders 테이블 컬럼 추가

- [ ] `estimatedReadyTime` (timestamp, nullable) - 예상 완료 시간 (픽업 시간)

**참고**:

- 취소 기능이 없으므로 `cancelReason` 필드는 불필요합니다
- 포장 전용이므로 `deliveryType` 필드는 필요하지 않습니다
- 단순성을 위해 `specialRequests` (특별 요청사항) 필드도 제외합니다

### 새 테이블 생성

- [ ] `storeHours` (영업 시간 관리 - 필수)
  - `id` (uuid, PK)
  - `profile_id` (uuid, FK to profiles)
  - `dayOfWeek` (integer, 0-6, 0=일요일)
  - `openTime` (time)
  - `closeTime` (time)
  - `isClosed` (boolean, default false)

**참고**: 단순성을 위해 다음 테이블은 제외합니다:

- `customerFavorites` (즐겨찾기 기능)
- `orderReviews` (리뷰 시스템)
- `customerNotes` (고객 메모)

## 픽업 시간 계산 로직 설계

### 기본 로직 (단순 버전)

1. 현재 시간 확인
2. 영업 시간 확인
3. 예상 조리 시간 확인 (점주가 설정한 기본값 또는 메뉴별 조리 시간)
4. 계산: `현재 시간 + 예상 조리 시간 = 픽업 가능 시간`
5. 픽업 시간이 영업 시간 내인지 확인
6. 영업 시간을 벗어나면 다음 영업일 첫 시간으로 조정

### 향후 개선 가능 (단순성 유지를 위해 제외 권장)

- 주문 대기 수를 고려한 동적 조리 시간 계산
- 메뉴별 조리 시간 차이 반영
- 피크 타임 보정
