# pojang.one — 시장·노점 현금 미리주문 웨지 분석 (인덱스)

automation 프로젝트의 폐차장/FSM 니치 게이트 분석과 동일 포맷으로, pojang.one을 *시장·경쟁 게이트* 관점에서 점검한 패키지(2026-06-06).

## 배경
pojang.one = 포장주문 전용 웹앱(결제 미탑재·현장결제·식당별 URL + 카카오채널). 최초 점검에서 "수수료 없는 포장주문"은 카카오·네이버·무료QR 레드오션으로 **보류**였으나 — 점주가 직접 확인한 사실(**카카오 픽업은 선결제 필수, 현장 현금 불가**) + 타겟이 **시장상인·노점**임이 더해지며 **조건부 GO**로 재판정됨. 최초 점검 원본: `../시장경쟁_게이트_점검_2026-06-06.md`.

## 파일 (읽는 순서)
- **`06_종합판단.md`** ⭐ — 6축 통합 + 게이트(**조건부 GO**) + 3대 미검증 + 재정렬 권고. **여기부터.**
- `00_제품정의_포지셔닝.md` — 제품 as-is + to-be 재포지셔닝
- `01_시장규모.md` — 도달 ~20만 점포·TAM 연 240억(가설), 통계 사각
- `02_경쟁점유자.md` — 게이트 핵심: 현장현금·미등록층은 점유자 부재(카카오 선결제·영업신고증 / 무료QR 결제수수료 모델). "만나서 결제" 오해 정정
- `03_타겟페인_웨지.md` — 페인 + 적합/부적합 업종(노점 일반 제외)
- `04_규제_합법성.md` — 현금결제 합법 코어 vs 탈세 방조 위험 경계 + 제품 가드레일 3개
- `05_WTP_GTM.md` — 자부담 5~9천원·스마트상점 바우처 게이트·상인회 시장단위 온보딩
- `인터뷰_가이드.md` — 상인 5~7곳 검증 질문지 + 게이트 판독

## 핵심 판정 (한눈에)
- **조건부 GO** — 현장현금+영업신고증 무관의 구조적 공백 실재(정산/포트원과 달리 점유자가 못 옴).
- **3대 재정렬 필수**: ① 타겟 노점→**시장 단골 미리주문 적합업종**(반찬·떡·정육·족발·김밥) ② 수익 월단독→**스마트상점 바우처**(자부담 0~3천) ③ 카피 탈세→**현금OK·가입간편·단골 미리주문** + 합법 가드레일.
- **미검증(깨지면 보류)**: 고객 *미리주문 행동* / 상인 *WTP* / 합법 카피 매력.
- **다음**: 거점 시장 1곳 → 상인 인터뷰 → 시장단위 파일럿.

## ⚠️ 합법성
현금결제·간이과세층 정상기록은 합법. 단 "매출 은닉 도구화 + 탈세 광고"는 조세포탈 방조 위험 → `04` 가드레일 준수(주문 정상기록·현금영수증 안내·탈세 카피 금지).
