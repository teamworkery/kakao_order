## 2. "만나서 결제" 오해 정정 (최초 점검 오류)
- 카카오 마케팅의 "결제는 카카오페이, 카드는 물론 **만나서 직접 결제(현금/카드)**"는 **배달=요기요 연동의 라이더 결제** 맥락[확인, "주문하기 by 요기요"].
- 실제 픽업/포장 체크아웃(톡체크아웃) 결제수단 = 카카오페이 카드/머니/포인트·일반카드·휴대폰결제·무통장입금 — **6종 전부 온라인 선결제, 현장 현금 없음**[확인, cs.kakao].
- → 사용자(점주) 직접 확인과 일치. **현장 현금결제는 카카오가 점유 못 한 빈칸.**

## 3. 게이트 분석
- **Q1 핵심 컨셉(카톡 포장주문) 점유? → 예**, 단 *선결제·사업자 전제* 조건부.
- **Q2 차별점(재방문 쿠폰) 점유? → 예**(카카오 자체 제공). → 재방문 쿠폰은 해자 아님.
- **Q3 현장 현금결제 점유? → 아니오(빈칸).** 카카오·네이버·무료QR 모두 구조적으로 못 함.
- **Q4 미등록·영세상인 접근 점유? → 아니오(빈칸).** 입점요건·결제수수료 모델이 이 층을 배제.
- **정산(포트원)과의 차이**: 정산은 점유자가 *수기·외부등록까지* 다 먹어 틈이 없었음. 여기는 **점유자가 정책·수익모델상 못 들어오는 자리**가 존재 → 보류 아닌 조건부.

## 4. 다만 — 공백이 작고 행동이 미검증
- 이 빈칸의 *크기*(TAM)와 *고객 행동*(영세·현금 고객이 온라인 미리주문을 하나)은 별도 게이트(→ `01_시장규모.md`, `05_WTP_GTM.md`). 공백이 있다 ≠ 사업이 된다.
