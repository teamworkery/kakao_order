# 축0 — 제품 정의 & 포지셔닝 (grounding)

> 출처: 프로젝트 `marketing/marketing.pdf`, `CLAUDE.md`, 코드/배포 상태(2026-06-06).

## 1. 제품 (as-is)
- **pojang.one** — 포장주문 전용 웹 서비스. 식당별 고유 주문 URL(`pojang.one/가게이름`) + 카카오톡 채널 주문하기 연결.
- **정책(코드·문서 확인)**: 포장 전용(배달 X), **결제 미탑재**(현장 현금/카드 직접결제), 주문 후 취소 불가, 픽업 가능시간 계산·표시가 핵심 기능.
- **기술**: React Router 7(SSR) + Supabase(PostgreSQL/Auth) + Tailwind/Shadcn + n8n(SMS/알림톡). Vercel 배포(`www.pojang.one`), RLS 적용 완료.
- **현황**: 실가게 1곳(굿모닝차이나·부천 상동) 시드·가동, 베타 5팀 모집. 미완 = 카카오 알림톡 실발송·커스텀 SMTP·OAuth prod 검증.

## 2. 포지셔닝 (마케팅 PDF as-is)
- 표면 가치제안: "배달앱과 달리 **수수료 0**, 월 15,000원 구독, 고객 현장결제."
- 차별 주장: 카카오채널 **양방향 소통·재방문 쿠폰**(이벤트/비오는날/당첨확률).
- 베타 5팀 한정 이벤트(3개월 혜택 + 카카오채널 개설 대행 무료).

## 3. 포지셔닝 재정의 (이 분석의 결론 — to-be)
- **as-is의 문제**: "수수료 없는 포장주문"은 카카오·네이버·무료QR이 점유한 레드오션(→ `02_경쟁점유자.md`). 이 카피로는 정면 충돌.
- **진짜 차별축 2개(경쟁자가 구조적으로 못 가짐)**:
  1. **현장 현금결제 + 선결제 강제 없음** — 카카오/네이버 픽업은 온라인 선결제 필수.
  2. **영업신고증·사업자·카드단말기 없이 URL 하나로 즉시** — 플랫폼 입점요건을 우회.
- **to-be 타겟**: 일반식당이 아니라 **전통시장 내 단골 미리주문 적합 영세점포**(반찬·떡·정육/생선·족발보쌈·김밥도시락). 노점(충동구매 군것질)은 제외(→ `05_WTP_GTM.md`).
- **to-be 수익**: 월 단독과금(15,000원)은 영세층에 높음 → **스마트상점 SaaS 바우처**에 끼워 자부담 0~3천원으로(→ `05`).
- **to-be 카피 원칙**: "탈세"가 아니라 "**카드단말기 없이 단골 미리주문·현금 OK·가입 간편**"으로만 소구(→ `04_규제_합법성.md`).
