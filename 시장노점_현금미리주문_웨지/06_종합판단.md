# pojang.one 시장·노점 현금 미리주문 웨지 — 종합판단

> **조사일**: 2026-06-06 · **방식**: 6축(제품·시장·경쟁·페인·규제·WTP/GTM) + 경쟁 결제정책 웹검증
> **한 줄 결론**: **조건부 GO(타겟·수익·카피 재정렬 전제).** "일반식당 탈수수료 포장"은 레드오션이나, **현장현금+영업신고증 무관**의 빈칸은 카카오·네이버·무료QR이 *구조적으로 못 들어오는 진짜 공백*. 단 **노점→시장 단골 미리주문 적합 업종으로 좁히고**, **월 단독과금→스마트상점 바우처로 끼우고**, **탈세→편의 카피로 바꾸면** 검증할 가치 있음. 핵심 미검증 = 고객 행동·WTP.

## 2. 게이트 판정 — 조건부 GO
- ✅ **공백 실재**: 정산(포트원 완전점유)과 달리, 점유자가 정책·수익모델상 못 오는 자리가 있음.
- ✅ **합법 코어**: 현금결제·간이과세층 정상 기록은 합법(가드레일 준수 시).
- ✅ **수익·진입 경로**: 바우처 + 상인회 시장단위 온보딩.
- ⚠️ **3대 미검증(이게 진짜 게이트)**:
  1. **행동** — 적합업종(반찬·떡·정육·족발·김밥) 단골이 *온라인 미리주문*을 실제로 하나? 안 하면 GO 무효.
  2. **WTP** — 바우처 끼워도 자부담을 내나? 영세 가격민감 극심.
  3. **카피/합법** — 탈세 소구 없이도 매력적인가(현금 OK·가입 간편·단골 미리주문).

## 3. 권고 — 3가지 재정렬 후 검증
1. **타겟 재정렬**: 노점 일반 ✗ → **전통시장 내 단골 미리주문 적합 점포**. 베타 5팀을 이 렌즈로 재분류(일반식당이면 약함).
2. **수익 재정렬**: 월 15,000원 단독과금 ✗ → **스마트상점 SaaS 바우처 게이트**(자부담 0~3천원). 단독 카피 금지.
3. **카피 재정렬 + 가드레일**: "탈세/수수료 회피" ✗ → "**카드단말기 없이 단골 미리주문·현금 OK·가입 간편**". 제품에 ①현금주문 정상기록(은닉옵션 금지) ②현금영수증 발급안내 ③탈세 카피 금지 내장.

## 4. 다음 단계 (검증부터, 폐차장 패턴)
1. **거점 시장 1곳 선정** — 먹거리 비중 높고 디지털 지원사업 선정 이력 있는 전통시장(상인회 활성).
2. **상인 5~7곳 인터뷰** — `인터뷰_가이드.md`(영업신고증 유무·카카오/네이버 안·못 쓰는 이유·단골 미리주문 행동·바우처 자부담 WTP).
3. **시장 단위 파일럿** — 상인회 통해 적합업종 점포 N곳 동시 온보딩, 단골 미리주문 전환 실측.
4. **게이트**: 행동(미리주문 전환) 또는 WTP(바우처 자부담)가 깨지면 → 웨지 보류·재배치. 둘 다 서면 → 카피·바우처 등록·시장 거점으로 정식 추진.
